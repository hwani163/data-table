## 요구사항별 결과

| Req | 상태 | 무엇이 들어갔나 (API) |
|---|---|---|
| **#1** 풍부한 표시 셀 (배지/태그/상태/코드) | ✅ 구현 | `column.display` — `{kind:'badge'}` · `{kind:'tags'}` · `{kind:'code'}` · `{kind:'text', color}`. 내부적으로 glide 공식 **TagsCell** + Text+`themeOverride` 로 환원(직접 canvas draw 안 함). 톤은 `BadgeTone`(success/warning/destructive/info/secondary/default) |
| **#2** 진짜 행 클릭 | ✅ 구현 | `onRowClick(row)` (셀/컬럼 무관) + `column.disableRowClick`. 액션 컬럼은 자동 제외 |
| **#3** 액션 컬럼(버튼/스위치) | ⚠️ 부분 구현 | `column.actions: (row)=>RowAction[]`, `RowAction={label,onClick,tone?,disabled?}` — 신규 canvas custom cell + posX 히트테스트. 토글은 `display:{kind:'toggle'}`(네이티브 체크박스). **단일 버튼·토글까지.** 오버플로 메뉴/확인 다이얼로그/스피너는 미포함 |
| **#5** inline edit + per-cell 스타일 | ✅ 구현 | `editable`+`onCellEdited`(기존) + 신규 `column.cellTheme: (value,row)=>Partial<Theme>` (셀별 배경/글자색/폰트) |
| **#6** pagination/loading/empty | ✅ 구현(에르고노믹스) | `loading`(오버레이) · `emptyMessage` · `footer`(상태바 위 슬롯, 페이저 꽂는 자리). 가상화 특성상 페이저 자체는 footer 에 직접 구성 |
| **#4** 확장/마스터-디테일 행 | ❌ 미구현(의도적) | glide 는 플랫 가상화 그리드. 행 사이 임의 React 패널은 가상화를 깨야 함 → 도구 부적합. 해당 테이블은 shadcn 유지 권장 |

## 신규 공개 API (v0.2)

```ts
// 컬럼에 추가된 필드
type DataTableColumn<T> = {
  // ...기존(accessorKey/header/size/editable/format/selectOptions 등)
  display?: DisplaySpec<T>;                              // 선언적 표시 셀 (format 보다 우선)
  actions?: (row: T) => RowAction[];                     // 액션 버튼 컬럼 (자동 disableRowClick)
  cellTheme?: (value: unknown, row: T) => Partial<Theme>;// 셀별 themeOverride
  disableRowClick?: boolean;                             // onRowClick 제외
};

type DisplaySpec<T> =
  | { kind: 'badge'; tone?: BadgeTone | ((v,row)=>BadgeTone); label?: (v,row)=>string }
  | { kind: 'tags';  values: (row)=>readonly string[]; tone?: BadgeTone | ((tag,row)=>BadgeTone) }
  | { kind: 'code' }
  | { kind: 'text';  color?: (v,row)=>string }
  | { kind: 'toggle' };

type BadgeTone = 'default'|'secondary'|'success'|'warning'|'destructive'|'info';
type RowAction = { label: string; onClick: () => void; tone?: BadgeTone; disabled?: boolean };

// 컴포넌트에 추가된 props
<DataTable
  onRowClick={(row) => ...}     // disableRowClick 컬럼은 제외
  loading={bool}
  emptyMessage={<...>}
  footer={<Pager .../>}
/>
```

셀 종류 우선순위: `actions` → `display` → select(dropdown) → `format` → 기본(number/text).

## canvas 제약 (정직하게)

마이그레이션 시 아래는 **기능이 1:1로 안 옮겨지고 대체/생략**됩니다:

- **셀 안 아이콘(lucide)** — canvas 라 못 그림. 상태 아이콘·파일타입 아이콘·health dot →
  `BadgeTone`(색)으로 의미 전달. 액션 버튼도 아이콘 대신 **짧은 텍스트 라벨**.
- **2줄 셀**(제목+설명 stack) — 1줄 텍스트만. 필요하면 컬럼 분리 권장.
- **삭제 확인 `<AlertDialog>`** — `RowAction` 에 내장 confirm 없음. `onClick` 안에서
  호출 측이 자체 다이얼로그를 띄우면 동작은 동일.
- **오버플로 `<DropdownMenu>` · 툴팁 · `<Switch>` 비주얼** — 미지원(토글은 체크박스).

이게 핵심인 테이블(특히 `rawdata-test`, `proxy-manager`)은 shadcn 유지가 맞습니다.
라이브러리의 sweet spot은 여전히 **dense · value-oriented 그리드**입니다 — 이번엔 거기에
"액션 컬럼 + 상태 배지" 정도가 더해진 app-shell 테이블까지 커버 범위를 넓힌 것.
